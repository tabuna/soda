# Breathing Metrics: объективная оценка

Чёткая таксономия, избыточность и однозначные рекомендации.

---

## Проблема: метрики похожи

Все 6 метрик связаны с «читаемостью», но измеряют **разные вещи** с разной степенью **пересечения**:

```
                    ┌─────────────────────────────────────┐
                    │           CBS (агрегат)              │
                    └─────────────────────────────────────┘
                                      │
         ┌────────────────────────────┼────────────────────────────┐
         │                            │                            │
         ▼                            ▼                            ▼
┌─────────────────┐      ┌─────────────────────────┐      ┌─────────────────┐
│  Когнитивная     │      │  Визуальный «воздух»    │      │  Именование     │
│  нагрузка       │      │  (избыточность!)        │      │                 │
├─────────────────┤      ├─────────────────────────┤      ├─────────────────┤
│ LCF (flow)      │      │ VBI ←→ COL              │      │ IRS             │
│ WCD (tokens)    │      │ оба используют blanks   │      │                 │
└─────────────────┘      └─────────────────────────┘      └─────────────────┘
```

---

## Три независимых измерения

| Измерение | Метрики | Уникальность |
|-----------|---------|--------------|
| **1. Control flow** | LCF | Единственная про nesting/conditions/loops |
| **2. Naming** | IRS | Единственная про длину идентификаторов |
| **3. Visual air** | VBI, COL | **Дублируют друг друга** — оба используют blank lines |

---

## Избыточность: VBI vs COL

| | VBI | COL |
|---|-----|-----|
| **Формула** | `(blanks/nLines) × blockUniformity` | `(blanks + shortBlocks + bonus) / nLines` |
| **Blanks** | ✓ | ✓ |
| **Дополнительно** | Однородность размеров блоков (σ) | Короткие блоки (≤3 строки) |
| **Вывод** | Измеряют одно и то же (визуальный «воздух») разными способами |

**Рекомендация:** Оставить **одну** — COL. Причина: формула проще, short blocks — понятная связь с SRP/extraction.

---

## Итоговая матрица: что оставить

| Метрика | Измеряет | Доказуемость | Решение |
|---------|----------|--------------|---------|
| **LCF** | Flow complexity | Исследования + SonarQube/clang-tidy | **Оставить** |
| **IRS** | Длина имён | Brains-on-Code, Sciencedirect | **Оставить** |
| **COL** | Blanks + short blocks | Style guides (PEP 8, ESLint) | **Оставить** (единственная «air») |
| **VBI** | Blanks + block uniformity | То же, что COL | **Убрать** (дублирует COL) |
| **WCD** | Token density | Эвристика, веса не обоснованы | **Опционально** или убрать |
| **CBS** | Агрегат | — | **Оставить** как dashboard |

---

## Чёткий вердикт

### Доказуемы и уникальны (оставить без вопросов)
1. **LCF** — nesting depth и cognitive complexity общепризнаны.
2. **IRS** — длина имён влияет на comprehension (14–19% в исследованиях).

### Доказуемы, но избыточны (оставить одну)
3. **COL** — blanks + short blocks. VBI убрать как дубликат.

### Недоказуемы (опционально)
4. **WCD** — веса 1.0/1.2/1.5/2.0 и пороги 10/20/35 — калибровка, не исследования.

### Агрегат
5. **CBS** — удобен для отчёта. Обосновывать через LCF, IRS, COL.

---

## Одна фраза для каждой метрики

| Метрика | Как доказать |
|---------|--------------|
| LCF | «Cognitive Complexity в SonarQube, nesting depth — стандарт» |
| IRS | «Brains-on-Code: descriptive names +14% к скорости поиска дефектов» |
| COL | «PEP 8, ESLint: blanks между блоками, short blocks = SRP» |
| WCD | «Эвристика: плотные строки тяжелее. Пороги — калибровка.» |
| CBS | «Сводный показатель. Обоснование — через LCF, IRS, COL.» |

## Рекомендуемая конфигурация (только достоверные метрики)

```json
{
  "rules": {
    "breathing": {
      "min_code_breathing_score": 25,
      "min_visual_breathing_index": 0,
      "min_identifier_readability_score": 75,
      "min_code_oxygen_level": 25
    },
    "complexity": {
      "max_logical_complexity_factor": 35,
      "max_weighted_cognitive_density": 0
    }
  }
}
```

`min_visual_breathing_index: 0` — не проверять VBI отдельно (дублирует COL).  
`max_weighted_cognitive_density: 0` — не проверять WCD (эвристика без исследований).
